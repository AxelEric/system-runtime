    This module is the main module of Runtime.
    It inits Runtime metamodel and loads Runtime core system.

    This module provides the following classes:

    • runtime
    • runtime-behavior
    • runtime-component
    • runtime-db
    • runtime-helper
    • runtime-log
    • runtime-metamodel
    • runtime-state
    • runtime-system
    • runtime-workflow
    • RuntimeDatabaseCollection
    • RuntimeError

    This module is a rollup of the following modules:

    • runtime-behavior
      This module manages the behaviors of all components. A behavior is a mecanism that allow users to add actions that will be executed when a specific state of a component will change.
    • runtime-component
      This module manages the components. It is the factory of all the components that are created by Runtime.
    • runtime-db
      This module manages Runtime database.
      Runtime database is a micro NoSQL Database that contains:
      - collections to store documents (schemas, types, components, ...) and
      - APIs to import or export documents.
      Runtime Database is closely linked to Runtime metamodel and Runtime components because:
      - all operations done by Runtime database must be compliant with the model before being finished,
      - insert operation automatically creates a component and
      - remove operation automatically destroy a component.
    • runtime-helper
      This module contains all the functions used by all the modules.
    • runtime-log
      This module contains all the functions that write a log.
    • runtime-metamodel
      This module manages Runtime metamodel.
      Runtime metamodel loads schemas and types, analyzes them and creates the component classes and related RuntimeDatabaseCollections.
    • runtime-state
      This module manages the states of all thecomponents.
    • runtime-system
      This module contains Runtime core system.
    • runtime-workflow
      This module manages the workflow of Runtime. It behaves like a workflow engine.
      It checks if the change of status of a component is valid to be executed. By valid, it means that:
      - the state is valid for the component,
      - the input (i.e. parameters) of all actions for the state are compliants with the model and
      - the output of all actions are compliants with the model.
      If an error occurs, the workflow will call the error state of the component and runtime.
      If the error can break the consistency of the current system, the worklow will stop.
