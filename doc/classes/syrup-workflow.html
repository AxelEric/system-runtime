    syrup-workflow

    Module: syrup-workflow
    Parent Module: syrup

    This module manages the workflow of syrup. It behaves like a workflow engine.
    It checks if the change of status of a component is valid to be executed. By valid, it means that:

    • the state is valid for the component,
    • the input (i.e. parameters) of all actions for the state are compliants with the model and
    • the output of all actions are compliants with the model.

    If an error occurs, the workflow will call the error state of the component and syrup.
    If the error can break the consistency of the current system, the worklow will stop.

    state
    (
    • params
    )

    Defined in src/workflow.js:725

    Change the state of a component.

    Worklow:

    1. Check if the component has not been destroyed
    2. Check if the state is a method or an event
    3. Search if there is a behavior with an action for the new state
    4. If so, get the action(s)
    5. Check if the conditons on input are compliant with the metamodel
    6. Call the action(s)
    7. If not an of event, check if the conditons on input are compliant with the metamodel
    8. If all is ok, the state of the component is updated
    9. Return the result

    Parameters:

    • params Object

      params to change the state
      {String} component id of the component
      {String} state state of the component
      {Array} data parameters to send to the action
